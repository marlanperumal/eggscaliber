# Figma Setup Checklist

## File Structure

- Product Screens (prototype flows)
- Design System (tokens and shared components)
- Experiments (A/B and exploratory concepts)

## Naming Conventions

- Use feature-prefix naming for pages and frames.
- Keep component naming aligned with code component names.

## Required States

For each key flow, include:

- Empty
- Loading
- Error
- Success

## Handoff Rules

- Link Figma frame URLs in Linear spec issues.
- Include expected behavior notes for interaction edge cases.
- Flag unresolved design questions before Build Ready gate.
