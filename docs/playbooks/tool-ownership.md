# Tool Ownership Model

## Linear

- Product roadmap and initiative status.
- Stage-gate state and risk tracking.
- User feedback and pilot decisions.
- Mandatory links to spec, design, and validation docs for each feature issue.
- Feature branch naming reference and PR linkage.

## GitHub

- Source code, pull requests, CI status, and releases.
- Engineering tasks linked to Linear items.
- Branches should be ticket-linked (for example `ENG-123-feature-name`).
- Do not merge or update `master` without explicit user approval.

## Figma

- Design artifacts and prototype states.
- Design system references.

## Cursor

- Agent orchestration and implementation execution.
- Local planning and coding workflows.

## Repository Docs

- Specs, ADRs, patterns, testing standards, and playbooks.
- Spec/design/validation artifacts are version controlled for each feature.
- Doc addendum sections are used for implementation-driven changes.
